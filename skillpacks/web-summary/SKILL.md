# Web Summary Skill

ใช้ skill นี้เมื่อผู้ใช้ต้องการสรุปข้อมูลจากเว็บไซต์ผ่าน URL โดยตรง
เช่น "สรุปเว็บนี้ให้หน่อย" หรือ "ช่วยสรุปข่าวจากลิงก์นี้"

## Workflow

1. ตรวจสอบว่าได้รับ URL แบบ `http/https` ครบถ้วน
2. ใช้ `web_fetch` เพื่อดึงเนื้อหาเว็บ
3. ถ้าเนื้อหายาวหรือเป็นหน้า index ให้สรุปเฉพาะสาระสำคัญ
4. ส่งสรุปแบบโครงสร้างสั้น อ่านง่าย

## Output Format (default)

- `Summary:` สรุปภาพรวม 3-6 บรรทัด
- `Key Points:` bullet 3-7 ข้อ
- `Actionable Notes:` สิ่งที่ผู้ใช้ควรรู้/ทำต่อ (ถ้ามี)

## Quality Rules

- ห้ามแต่งข้อมูลที่ไม่มีในหน้าเว็บ
- ระบุข้อจำกัดถ้าเนื้อหาไม่ครบหรือถูกตัด
- ถ้าหน้าเว็บเข้าถึงไม่ได้ ให้บอกสาเหตุและเสนอให้ลอง URL อื่น
- หากผู้ใช้ระบุภาษา ให้สรุปตามภาษาที่ผู้ใช้ต้องการ

## Tips

- ถ้าผู้ใช้ต้องการสรุปหลายลิงก์ ให้ทำทีละลิงก์แล้วค่อยรวมเปรียบเทียบ
- ถ้าผู้ใช้ต้องการรายละเอียดเชิงลึก ให้เสนอรอบที่สองแบบ section-by-section
